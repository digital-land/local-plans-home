How to set up automated consultation responses by email

This user guide sets out how to install email automation for Microsoft Outlook. It automatically saves emails in response to a Local Plan consultation as a database on Sharepoint.

From:
Department for Levelling Up, Housing and Communities
Published
23 August 2013

Create a new Sharepoint site

You need a Sharepoint site to store the email data. If your team already has one set up and you’d prefer to use that, you can skip this section and install the automation.

  1. Sign in to Sharepoint, then select Create site on the start page
  2. Select Team site and then Standard team template
  3. Select Use template on the preview page
  4. Give your site a name. For example, [Your LPA name] Local Plan Consultation responses
  5. Add a description for your Sharepoint site
  6. Assign an owner (this can be you or anyone else who will have access to your site)
  7. If you see the below screen, set the privacy settings as Private - only members can access this site, then choose your language and timezone
  8. Add users to your site using their email addresses. Anyone you want to have access to the Consultation responses will need access to the Sharepoint site

You should create a new Sharepoint site for each consultation you’re running. For example, if you want to set this automation for multiple mailboxes, each mailbox will need its own Sharepoint site.

Install the automation

1. Add the flow to Microsoft 365

  • Download this zip file by following the link
  • Go to make.powerapps.com in your browser and sign in (if prompted). If this is your first time using PowerApps, you may need to fill some contact information based on your preferences
  • Check the Environment option at the top of the screen and see if you can choose “Personal Productivity”. If so, select it. If not, leave your environment as the default option
  • Select Solutions from the left hand side menu, then Import Solution
  • Select the Browse button and find the zip file in your downloads folder – select the whole zip file (called Planningconsultationemailprocessing_1_0_0_13_managed), not individual items within it
  • Select Next to add the automation to Power Platforms

2. Connect the automation to your mailbox

  • You may be prompted to update some settings – select Continue
  • Before selecting Next, select the three dots beside Planning Consultation Mailbox and select Add new connection
  • Select the account that has access to your shared mailbox. You’ll be prompted to sign in to this mailbox – you’ll need the password in order to do this. If you don’t have the password to this mailbox, contact your Microsoft Administrator to find out how to get it.
  • Select Next to continue

3. Connect the automation to Sharepoint

You do not need to create a new folder for this automation. The Sharepoint list name field on this page (explained later in this step) will do this automatically.

Department for Levelling Up, Housing and Communities

The following steps may be in a slightly different order depending on your system.

  • Add the URL of the Sharepoint site you created earlier to the Sharepoint site URL field. It should be in the format: https://[orgname].sharepoint.com/sites/[site name] – for example, https://repscouncil.sharepoint.com/sites/repsconsultation.

    If you’re using an existing site, add its URL in the format https://[orgname].sharepoint.com/sites/[site name] – for example https://repscouncil.sharepoint.com/sites/repsconsultation – to the Sharepoint site URL field
  • Choose an appropriate list name (for example, [your LPA name] Consultation responses) and add it to the Sharepoint list name field.

    Your list name will be displayed on the left hand side navigation of your Sharepoint site once the automation is installed, so you can access it easily
  • Choose which mailbox folder you’d like the automation to run from – this will usually be the Inbox, but if you’ve set up rules to automatically forward responses to a different folder, you can change it
  • You can also turn on automatic responses. These are pre-written automatic replies to emails sent to your consultation inbox. Responses are sent once a day in bulk. To turn these on, select Yes for Email Acknowledgement Required
  • Enter the shared mailbox email address in the field Shared mailbox address

4. Import the automation

  • Select Import and wait until the “Currently importing” message at the top of your screen turns into a green bar, which tells you the import has been successful
  • Select the Managed button at the top of the list

The automation should now be installed, and you can test if it works.

Run the automation

To check that the automation has installed and works properly, start by sending a test email, including an attachment, to the shared mailbox address.

Make sure you do not have any unread emails in your shared mailbox before you start the test.

Once you’ve sent the email, follow these steps:

  • go to make.powerapps.com
  • go to Solutions on the left hand navigation pane
  • choose Managed at the top of the list
  • select Planning consultation email processing
  • select Run at the top of the screen, and the Run Flow button that will appear in the sidebar
  • select Done

This will manually run the automation. You can do this whenever you need to test it, or if you’d like to pull emails into the database outside the normal daily run time.

You should be able to see the run history on your Power Platforms page. The run request you’ve just made will say “Running”.

Use the Refresh icon if there is a delay between the run completing and the status automatically updating.

Once the run request has completed, the status should say “Succeeded”.

You can now go back to the Sharepoint site you linked to the automation.

Select the Settings/gear icon, select site contents and select the list name you specified in the Sharepoint list name field during setup. You should see a table with the following columns:

  • reference
  • email address
  • status
  • date and time
  • subject line
  • email link
  • email text
  • attachments

If your automation is working, you should see the data from your test email in the table, along with any other unread emails in your shared mailbox.

If you selected automatic responses, you should also receive an email acknowledgement back to your email address.

If the automation isn’t working, or the run request says “Failed” next to it, select the run request.

You’ll see a flow chart, which should have green ticks against every part of the flow that was successful. Finding the parts of the flow chart that were not successful should help you troubleshoot the issue. If you need further support, reach out to your IT team.

For information on how to delete the solution and content created during this test please go to the Uninstalling the automation section.

Using the automation for multiple consultations

If you want to use the solution across multiple consultations for one Local Plan, turn the automation back on when consultation reopens. The responses will be added to the same list as for your previous consultation.

If you want to have different lists and folders for each consultation, you need to remove the automation and install it again.

Access and use the database

If you’ve successfully tested the automation and everything is working properly, you should now be able to use the database.

To get started:

  1. go to your Sharepoint URL
  2. select Settings – the gear icon – and then select Site contents
  3. there will be a folder called ‘Planning Consultation Emails’ –  select this folder to open it
  4. there will be a sub-folder with the name of the list you created during the set-up process. All emails are stored in this folder
  5. update the status of any downloaded email by selecting the individual row the email is on
  6. a new panel will appear with all the details of that specific email, and you can edit the status from this panel

Export your data

  1. go to your Sharepoint site and select Export at the top of the page
  2. choose Export to Excel and download the file
  3. open your file in Excel

You’ll be able to see most of the email data, including the location in Sharepoint where the email is saved, but you will not be able to see any attachment data except the number of files that were attached.

Updating your exported database

Once you export the database, you can still pull new emails through to it by following these steps:

  1. open your downloaded Excel spreadsheet
  2. select the Data tab at the top of the Excel spreadsheet
  3. select Refresh all. This will bring any new emails that are in the database through to your Excel spreadsheet
  4. select Save once you’ve done this so you do not lose any data

Turn the automation off

When you’re no longer accepting responses to your consultation, make sure to turn off the automation.

  • go to make.powerapps.com
  • go to Solutions
  • choose Managed, then select Planning consultation email processing
  • select the Turn off button – if this has been successful, it will turn into a Turn on button

Uninstall the automation

If for any reason you no longer want to use the automation, or you need to reinstall it because of an issue, you can follow these steps.

To uninstall:

  1. go to make.powerapps.com and select Solutions
  2. choose Managed at the top of the list, then select Planning consultation email processing
  3. select the three dots next to ‘Planning consultation email processing’ and select Delete
  4. confirm you would like to remove this solution – once you’ve done this, it will remove the solution from your PowerApps environment
  5. to remove the lists created by the automation, navigate to your Sharepoint URL
  6. Select settings – the gear icon – then select Site contents
  7. in the Site contents page there will be a folder called Planning Consultation Emails. Select the three dots next to this and select Delete. This will remove all sub-folders contained within this folder
  8. on the same screen, select the three dots next to the list name you created during set-up, then select Delete. This will remove all sub-folders contained within this list
